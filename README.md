# Stephen Salyer — Portfolio

## Deploy to GitHub Pages in 5 minutes

### First time setup

1. Go to [github.com](https://github.com) and sign in (or create a free account)
2. Click **+** → **New repository**
3. Name it exactly: `stephen-salyer-portfolio` (or any name you want)
4. Set to **Public**
5. Click **Create repository**

### Upload your files

6. On the new repo page, click **uploading an existing file**
7. Drag ALL files from this folder into the browser window:
   - `index.html`
   - `uxElle.webp`
   - `Element.webp`
   - `variable-alias-flow-plugin.webp`
   - `Design-system-contrast-checker-figma-plugin.png`
   - `Login_-_Desktop-example-farmer-ecommerce.webp`
   - `dart-financials-dashboard.webp`
   - `doughnaughty-freelance-work.webp`
   - `Me-stephen-salyer.webp`
8. Click **Commit changes**

### Enable GitHub Pages

9. Go to **Settings** tab of your repo
10. Click **Pages** in the left sidebar
11. Under "Source" select **Deploy from a branch**
12. Select branch: **main**, folder: **/ (root)**
13. Click **Save**

Your site will be live at:
`https://YOUR-GITHUB-USERNAME.github.io/stephen-salyer-portfolio/`

GitHub usually takes 1-2 minutes to deploy.

### Custom domain (optional)

If you want `stephensalyer.com` to point here instead of Squarespace:
- In GitHub Pages settings, add your custom domain
- Update your domain's DNS to point to GitHub's servers
- GitHub will handle the SSL certificate automatically
